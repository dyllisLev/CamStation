# DAEMON GUIDE

## OVERVIEW
`cmd/camstationd/` is the composition root: flags/env, SQLite open/migrate, worker wiring, HTTP routes, static embed, and go2rtc proxy.

## WHERE TO LOOK
| Task | Location | Notes |
|------|----------|-------|
| Startup flags/env | `main.go` | `CAMSTATION_*` defaults live near `main`. |
| Route changes | `main.go` `routes(...)` | API handlers are currently registered in one mux. |
| Static frontend serving | `main.go` `spaHandler(...)` | Serves embedded `cmd/camstationd/web`. |
| go2rtc player proxy | `main.go` `go2RTCProxy(...)` | Keep allowlist narrow. |
| Console/live regressions | `main_test.go` | Tests inspect embedded source and route behavior. |
| Embedded assets | `web/` | Generated by `cd web && npm run build`; do not hand-edit. |

## CONVENTIONS
- `main.go` wires store, stream, recorder, cleanup, and camera packages; keep package logic out of route handlers when it belongs in `internal/`.
- `/` serves the control room console; `/live` remains the live monitoring workspace.
- `/player/` is the only public path into the local go2rtc player surface.
- Startup recovery runs before recorder workers start; preserve that order.
- Automatic cleanup is enabled only when max storage is configured.
- Recording workers auto-start only when `-recording-enabled` or `CAMSTATION_RECORDING_ENABLED=true` is set.

## ANTI-PATTERNS
- Do not expose raw go2rtc status APIs such as `/api/streams` through CamStation.
- Do not add public route responses that include raw RTSP credentials.
- Do not hand-edit files under `cmd/camstationd/web/assets`; rebuild from `web/`.
- Do not make route changes without checking whether frontend API types and query hooks need the same change.

## VERIFY
```bash
go test ./cmd/camstationd
go build -o camstationd ./cmd/camstationd
curl http://127.0.0.1:18080/api/health
```
