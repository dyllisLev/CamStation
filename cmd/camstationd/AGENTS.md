# DAEMON GUIDE

## OVERVIEW
`cmd/camstationd/` is the daemon composition root and controller layer: startup wiring, domain route files, SPA serving, safe go2rtc proxy, and backup scheduling.

## WHERE TO LOOK
| Task | Location | Notes |
|------|----------|-------|
| Startup flags/env | `main.go` | DB migrate, recovery, workers, backup scheduler, server lifecycle. |
| Route assembly | `routes.go` | `routeDeps` and domain route registration order. |
| Core/static/proxy | `routes_core.go`, `spa_proxy.go` | Health, SPA fallback, `/player/` allowlist. |
| Cameras/profiles | `routes_cameras.go`, `routes_camera_*.go` | Secret hygiene and go2rtc restart warnings. |
| Recording APIs | `routes_recordings.go`, `routes_recordings_dto.go` | Storage, workers, segments, play/download URLs. |
| Backup APIs | `routes_backup.go`, `backup_scheduler.go` | Config, jobs, default recordings source, schedule ticks. |
| Alerts/settings/jobs | `routes_alerts.go`, `routes_settings_jobs.go` | Test alert, settings, generic job operations. |
| System/viewers/streams | `routes_system*.go`, `routes_viewers.go`, `routes_streams.go` | Diagnostics, maintenance, viewer registry, stream status. |
| Embedded assets | `web/` | Generated by `cd web && npm run build`; do not hand-edit. |

## CONVENTIONS
- `main.go` constructs packages; route files should stay thin and push reusable behavior into `internal/`.
- `routeDeps` is the only dependency bundle for handlers; add shared dependencies there.
- Public DTOs belong in `routes_public_dtos.go` or domain DTO files; keep secret/runtime redaction centralized.
- `/player/` is the only public path into the local go2rtc player surface.
- Startup recovery runs before recorder workers; backup scheduler starts before HTTP serving.
- The backup route defaults an omitted source to the managed recordings dir; callers should not need to send runtime paths.
- Tests are route-heavy. Add/adjust `routes_*_test.go` when API shape, redaction, or status behavior changes.

## ANTI-PATTERNS
- Do not expose raw go2rtc status APIs, raw RTSP inputs, webhook URLs, auth headers, or local filesystem paths.
- Do not hand-edit `cmd/camstationd/web/assets`; rebuild from `web/`.
- Do not add a route without checking matching frontend API types/query hooks.
- Do not let failed validation create a permanently active job.
- Do not restart/kill runtime processes from handlers except through owned managers.

## VERIFY
```bash
go test ./cmd/camstationd
go build -o camstationd ./cmd/camstationd
curl http://127.0.0.1:18080/api/health
```
